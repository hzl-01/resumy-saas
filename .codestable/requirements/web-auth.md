# Web 服务、AI 主入口与用户认证

## 用户故事

- 作为一个不懂命令行的求职者，我希望打开浏览器就能上传旧简历并生成岗位定制版，而不是安装 Node.js 敲命令。
- 作为一个想快速求职的人，我希望先给旧简历和目标岗位 JD，让 AI 先起草，而不是自己从空白表单开始填。
- 作为团队里的非技术成员，我希望把 docx/PDF 简历拖到网页上，让 AI 理解后直接进入可编辑草稿和 PDF 导出流程。

## 为什么需要

resumy 现有的 CLI 工具对于技术用户很好用，但对大部分人来说，"打开终端敲命令"这件事本身就劝退了。真正更自然的产品体验不是“先手填整份简历”，而是“上传旧简历 + 给目标岗位 JD + 让 AI 先起草，再由用户确认和微调”。

## 怎么解决

通过 `resumy serve` 命令启动一个 Web 服务。用户在浏览器里注册账号、登录后，以“上传旧简历 + 输入目标岗位 JD”为主入口，由 Eino / 模型驱动的 AI 先生成可编辑草稿，再由用户进入 editor 修订，最后生成并下载 PDF。所有数据保存在服务器上，下次登录还能接着改。

## 边界

- 不提供付费/订阅功能（MVP 免费）
- 不支持团队协作/共享简历
- 不提供简历投递/职位匹配
- 一期仅支持单语言
